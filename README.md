## 🚧 Projects

### ⚙️ DAG Workflow Orchestration Engine *(In Progress)*
> `Java` `Spring Boot` `PostgreSQL` `Kafka` `Redis` `Docker` `Multithreading`

Building a distributed workflow orchestration system from scratch — think a simplified Apache Airflow.

```
User submits DAG → System creates execution instance → Tasks run in dependency order

A (READY) → B (CREATED) → C (CREATED)
                ↓ A completes
A (COMPLETED) → B (READY) → C (CREATED)
```

**What's built:**
- `WorkflowDefinition` → `WorkflowExecution` → `TaskExecution` entity model
- DAG parsing & dependency resolution engine
- Status machine: `CREATED → READY → RUNNING → COMPLETED`
- REST APIs: `POST /workflow` · `POST /workflow/{id}/execute`

**Coming next:** Worker engine · Kafka task queue · Redis distributed locking · Retry logic · Monitoring
